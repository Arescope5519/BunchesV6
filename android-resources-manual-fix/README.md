# Manual Android Resources Fix

## Problem
Your Android build is failing because the `android/app/src/main/res` directory is missing required resources:
- `mipmap/ic_launcher` (app icons)
- `string/app_name`
- `style/AppTheme`
- `style/Theme.App.SplashScreen`

## Solution 1: Automated Regeneration (RECOMMENDED)

Run the PowerShell script from the project root:

```powershell
.\fix-android-resources.ps1
```

This will:
1. Backup your existing android directory
2. Clean Expo cache
3. Regenerate the android directory using `npx expo prebuild`
4. Verify all resources are present

## Solution 2: Manual Fix

If automated regeneration fails, manually copy these files:

### Step 1: Copy XML Resources

Copy all XML files from this directory to your android project:

```
android-resources-manual-fix/values/strings.xml
  → android/app/src/main/res/values/strings.xml

android-resources-manual-fix/values/styles.xml
  → android/app/src/main/res/values/styles.xml

android-resources-manual-fix/values/colors.xml
  → android/app/src/main/res/values/colors.xml

android-resources-manual-fix/drawable/splashscreen.xml
  → android/app/src/main/res/drawable/splashscreen.xml
```

### Step 2: Generate App Icons

The app icons need to be generated from your `assets/icon.png` file. You have two options:

#### Option A: Use Expo to generate icons
```bash
npx expo prebuild --clean
```

#### Option B: Generate icons manually

1. Use an online tool like https://icon.kitchen or https://easyappicon.com
2. Upload your `assets/icon.png`
3. Download the Android icon pack
4. Extract and copy all `mipmap-*` folders to `android/app/src/main/res/`

The required mipmap folders are:
- `mipmap-mdpi/` (48x48)
- `mipmap-hdpi/` (72x72)
- `mipmap-xhdpi/` (96x96)
- `mipmap-xxhdpi/` (144x144)
- `mipmap-xxxhdpi/` (192x192)

Each should contain:
- `ic_launcher.png`
- `ic_launcher_round.png`

### Step 3: Rebuild

After adding the resources, rebuild your app:

```powershell
cd android
.\gradlew clean
.\gradlew assembleRelease
```

## Why This Happened

The `android/` and `ios/` directories are automatically generated by Expo and are gitignored (see `.gitignore` lines 40-41). Sometimes the generation process can fail or become corrupted, leading to missing resources.

## Prevention

Always use `npx expo prebuild --clean` when:
- First setting up the project
- After adding new dependencies
- After updating Expo SDK version
- When resource errors occur
